# DNS Device Fingerprinting

A cybersecurity and machine learning project focused on identifying and profiling devices based on their DNS traffic behavior.

---

## Project Overview

This project was developed as part of a cybersecurity course and explores how DNS traffic can be used to create behavioral fingerprints for network devices.

The main objective was to investigate whether devices can be distinguished and re-identified based solely on their DNS query patterns, without relying on traditional identifiers such as IP addresses or MAC addresses.

The project combines concepts from:

* Cybersecurity
* Network Traffic Analysis
* Machine Learning
* Natural Language Processing (NLP)
* Data Visualization

---

## Objectives

* Analyze DNS traffic generated by different devices
* Build behavioral profiles based on DNS requests
* Transform DNS activity into machine learning features
* Measure similarities between devices
* Identify the most discriminative DNS patterns
* Explore device re-identification techniques

---

## Methodology

### Data Collection

The project starts from DNS traffic logs generated by multiple devices.

Each device is represented through its sequence of DNS requests, creating a behavioral signature that can later be analyzed and compared.

### Feature Engineering

Several techniques were applied to transform DNS traffic into machine learning features:

* DNS token extraction
* Behavioral sequence construction
* TF-IDF vectorization
* Frequency analysis
* Similarity metrics

### Similarity Analysis

To compare device behavior, the following techniques were used:

* Cosine Similarity
* Distance-based comparisons
* Behavioral clustering

### Machine Learning

Random Forest was employed to:

* identify important features
* evaluate discriminative DNS patterns
* understand which domains contribute most to device identification
